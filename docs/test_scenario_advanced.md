# 고수준 통합 도구 고급 테스트 시나리오

이 문서는 실제 Windows 사용자들이 자주 사용하는 프로그램들을 기반으로 한 고수준 테스트 시나리오를 정의합니다.

## 테스트 환경 준비

### 필수 프로그램 설치
- Microsoft Office (Word, Excel, PowerPoint) 또는 Office 365
- 웹 브라우저 (Chrome, Edge, Firefox 중 하나 이상)
- 파일 탐색기 (Windows 기본)
- 메모장 (Windows 기본)
- 계산기 (Windows 기본)
- Outlook 또는 다른 이메일 클라이언트 (선택적)

### 테스트 전 확인사항
1. 모든 테스트 대상 프로그램이 설치되어 있음
2. 테스트용 샘플 파일들이 준비되어 있음
3. 인터넷 연결이 되어 있음 (웹 브라우저 테스트용)

---

## 1. Microsoft Excel 테스트 시나리오

### 시나리오 1.1: 새 Excel 파일 생성 및 데이터 입력

**사용자 목표**: Excel에서 새 파일을 만들고 간단한 데이터를 입력하고 저장

**테스트 단계**:
1. Excel 실행 (시작 메뉴 또는 실행)
2. 새 통합 문서 생성
3. A1 셀에 "이름" 입력
4. B1 셀에 "나이" 입력
5. A2 셀에 "홍길동" 입력
6. B2 셀에 "30" 입력
7. 파일 저장 (Ctrl+S 또는 파일 메뉴)

**예상 LLM 명령**:
```
"Excel을 열고 새 파일을 만들어서 A1에 '이름', B1에 '나이', A2에 '홍길동', B2에 '30'을 입력하고 저장해줘"
```

**구현 예시**:
```json
{
  "name": "interact_window",
  "arguments": {
    "window_title": "Excel",
    "actions": [
      {"type": "click_text", "search_text": "A1"},
      {"type": "type", "text": "이름"},
      {"type": "key", "key": "tab"},
      {"type": "type", "text": "나이"},
      {"type": "key", "key": "enter"},
      {"type": "type", "text": "홍길동"},
      {"type": "key", "key": "tab"},
      {"type": "type", "text": "30"},
      {"type": "hotkey", "keys": ["ctrl", "s"]}
    ]
  }
}
```

**검증 포인트**:
- ✅ Excel이 정상적으로 열리는가?
- ✅ 셀에 정확히 입력되는가?
- ✅ 파일이 저장되는가?

---

### 시나리오 1.2: Excel에서 수식 입력 및 계산

**사용자 목표**: Excel에서 간단한 합계 수식을 입력하고 결과 확인

**테스트 단계**:
1. Excel 열기
2. A1~A5에 숫자 입력 (예: 10, 20, 30, 40, 50)
3. A6 셀에 SUM 수식 입력 (=SUM(A1:A5))
4. 결과 확인

**예상 LLM 명령**:
```
"Excel에서 A1부터 A5까지 10, 20, 30, 40, 50을 입력하고 A6에 합계 수식을 넣어줘"
```

**검증 포인트**:
- ✅ 수식이 올바르게 입력되는가?
- ✅ 수식 결과가 계산되는가?

---

### 시나리오 1.3: Excel 차트 생성

```

```

**검증 포인트**:

## 3. 웹 브라우저 테스트 시나리오

### 시나리오 3.1: 웹 검색 수행

**사용자 목표**: 브라우저에서 검색어를 입력하고 검색 결과 확인

**테스트 단계**:
1. Chrome/Edge 실행
2. 주소창 클릭 또는 검색창 클릭
3. 검색어 입력 (예: "Python 자동화")
4. Enter 키 입력 또는 검색 버튼 클릭
5. 검색 결과 확인

**예상 LLM 명령**:
```
"Chrome에서 'Python 자동화'를 검색해줘"
```

**구현 예시**:
```json
{
  "name": "interact_window",
  "arguments": {
    "window_title": "Chrome",
    "actions": [
      {"type": "click_text", "search_text": "주소"},
      {"type": "type", "text": "Python 자동화"},
      {"type": "key", "key": "enter"}
    ]
  }
}
```

**검증 포인트**:
- ✅ 브라우저가 정상적으로 열리는가?
- ✅ 주소창/검색창을 정확히 찾는가?
- ✅ 검색이 수행되는가?

---

### 시나리오 3.2: 웹사이트 로그인

**사용자 목표**: 웹사이트에 로그인

**테스트 단계**:
1. 브라우저에서 웹사이트 접속
2. 로그인 버튼/링크 클릭
3. 사용자명 입력 필드 찾기 및 입력
4. 비밀번호 입력 필드 찾기 및 입력
5. 로그인 버튼 클릭

**예상 LLM 명령**:
```
"GitHub에 로그인해줘. 사용자명은 [username], 비밀번호는 [password]야"
```

**주의사항**: 
- 실제 비밀번호는 보안상 직접 입력하지 말고, 테스트용 더미 계정 사용
- 민감한 정보는 환경 변수나 설정 파일에서 읽도록 구현

**검증 포인트**:
- ✅ 로그인 페이지를 찾는가?
- ✅ 입력 필드를 정확히 식별하는가?
- ✅ 로그인이 성공하는가?

---

## 알려진 제한사항 및 해결 방법

### 1. 동적 콘텐츠
**문제**: 웹페이지나 애플리케이션의 콘텐츠가 동적으로 변경됨
**해결**: 인덱싱 후 즉시 작업 수행, 또는 재인덱싱 수행

### 2. 다국어 지원
**문제**: 영어가 아닌 언어의 텍스트 인식 정확도
**해결**: OCR 언어 설정 확인, 필요시 언어별 인덱싱

### 3. UI 변경
**문제**: 프로그램 업데이트로 UI가 변경됨
**해결**: 정기적인 테스트 및 시나리오 업데이트

### 4. 네트워크 의존성
**문제**: 웹 관련 작업은 네트워크 상태에 의존
**해결**: 네트워크 상태 확인, 타임아웃 설정
